## 12 — Deviations

1. **System-level php.ini edit.** `extension=intl` was commented
   out in `C:\xampp\php\php.ini` line 934, blocking Filament
   install. With operator authorisation (Option B), uncommented
   the line. Single-line, reversible. Outside Laravel project
   scope — disclosed here so the next teammate's environment
   setup is reproducible.

2. **Spec field-name substitution in seeder.** Spec template
   used `phone_verified_at => now()`, but the schema has
   `is_verified_phone` (boolean). Substituted to match schema.
   Documented inline in `AdminUserSeeder.php` and §7 above.

3. **Existing `role` enum coexists with new `is_admin` boolean.**
   Phase 2.1 already added `role ENUM('customer','admin')`. D-4.1-2
   chose the boolean for Filament without addressing the
   redundancy. Both columns now encode admin status; Filament
   reads only `is_admin`. Documented in the migration's docblock
   and `ADMIN_SETUP.md`. A future commit can consolidate.

4. **Frontend full-suite re-run was non-formal.** Brief step 25
   only requires smoke for this backend commit; smoke is 3/3.
   Optional full e2e showed 24/25 with one auth-edges flake that
   is unrelated to this commit's surface area (no Phase 4.1
   change touches frontend code, API auth middleware, or the
   toast component). Flagged for future verification but not
   treated as a 4.1 regression.
